# Security Policy

## Reporting a vulnerability

Please do **not** open a public issue for security problems. Instead:

- Preferred: open a private advisory via the repository's [Security tab](https://github.com/GreptimeTeam/greptimedb-ingester-ts/security/advisories/new) (GitHub Security Advisories).
- Alternative: email `security@greptime.com` with a reproduction and impact assessment.

We aim to acknowledge within 3 business days and keep you informed as we investigate. Coordinated disclosure timelines are negotiated case by case based on severity and fix readiness.

## Supported versions

The latest `0.x` release receives security fixes. Once `1.0` ships, the current and previous minor will receive fixes; older releases will not be patched.

## Scope

This policy covers the `@greptime/ingester` package in this repository. Vulnerabilities in the GreptimeDB server itself should be reported through [greptimedb](https://github.com/GreptimeTeam/greptimedb/security/advisories).
